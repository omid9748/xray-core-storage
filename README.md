## 🌍 چرا این اسکریپت؟

سرورهای داخل ایران (مانند ابر آروان، ایران‌سرور، هرتزنر ایران و...) به دلیل فیلترینگ و تحریم‌های بین‌المللی، در دانلود پکیج‌ها، مخازن داکر (Docker)، پایتون (PIP) و اوبونتو (`APT`) دچار اختلال شدید هستند. 

این اسکریپت ترافیک کل سیستم‌عامل و مدیریت پکیج را به سمت هسته قدرتمند **Xray** هدایت می‌کند. ویژگی منحصربه‌فرد این اسکریپت این است که برای استخراج فایل‌ها، نیازی به اینترنت آزاد و دانلود ابزارهایی مثل `unzip` ندارد و این کار را کاملاً آفلاین به کمک پایتون داخلی لینوکس انجام می‌دهد.

---

## 📂 ساختار درختی پوشه‌ها پس از نصب

پس از اجرای کامل، چیدمان فایل‌های پروژه روی سرور شما به این صورت خواهد بود:

```text
/opt/xray/
├── xray              # فایل اجرایی و اصلی هسته Xray
├── config.json       # فایل کانفیگ شخصی شما
├── geoip.dat         # دیتابیس تشخیص آی‌پي کشورها
└── geosite.dat       # دیتابیس دسته‌بندی سایت‌ها

## 🛠 راهنمای استفاده قدم به قدم (فقط در ۳ دقیقه)

### 1️⃣ مرحله اول: آماده‌سازی هسته Xray روی سرور

ابتدا با دسترسی `root` وارد ترمینال سرور اوبونتو خود شوید و پوشه کاری را بسازید:

```bash
mkdir -p /opt/xray

```

سپس آخرین نسخه فایل `Xray-linux-64.zip` را دانلود کرده، نام آن را دقیقاً به **`xray.zip`** تغییر دهید و با برنامه‌های مدیریت فایل (مانند SFTP، MobaXterm، FileZilla یا پنل وب آروان) داخل مسیر `/opt/xray/` آپلود کنید.

### 2️⃣ مرحله دوم: دریافت و اجرای اسکریپت

دستور زیر را بزنید تا اسکریپت راه‌انداز (بدون متن فارسی جهت جلوگیری از بهم‌ریختگی ترمینال) از گیت‌هاب دانلود شود:

```bash
cd /opt/xray
wget -q "[https://raw.githubusercontent.com/omid9748/xray-core-storage/main/install.sh](https://raw.githubusercontent.com/omid9748/xray-core-storage/main/install.sh)" -O install.sh

```

حالا به اسکریپت مجوز اجرا داده و آن را استارت کنید:

```bash
chmod +x install.sh
sudo ./install.sh

```

### 3️⃣ مرحله سوم: چسباندن کانفیگ JSON

در اواسط اجرای اسکریپت، پیام دریافت کانفیگ ظاهر می‌شود. مراحل زیر را دقیق انجام دهید:

1. کانفیگ مشتری (Client JSON) خود را کپی کنید.
2. در ترمینال راست‌کلیک کنید تا کل متون پیست شود.
3. یک بار دکمه **Enter** را بزنید تا به خط خالی بعدی برود.
4. کلیدهای **`Ctrl + D`** را روی کیبورد فشار دهید تا تغییرات ذخیره و سرویس استارت شود.

## ⚠️ تنظیمات بسیار مهم کانفیگ (پیش‌نیاز پورت‌ها)

برای اینکه سیستم‌عامل اوبونتو بتواند ترافیک خود را به Xray بفرستد، حتماً باید بخش ورودی‌ها (`inbounds`) در فایل JSON شما شامل پورت‌های زیر باشد. اسکریپت به صورت پیش‌فرض سیستم را روی پورت **20809** تنظیم می‌کند تا با برنامه‌های دیگر تداخلی نداشته باشد:

| نوع پروتکل ورودی | پورت محلی (Local Port) | آی‌پي گوش‌به‌زنگ | توضیحات |
| --- | --- | --- | --- |
| **HTTP** | `20809` | `127.0.0.1` | **الزامی** (برای ترافیک APT و کل سیستم) |
| **SOCKS** | `2080` | `127.0.0.1` | اختیاری (برای ابزارهای تحت ساکس) |

## 🔄 مرحله نهایی و تست پرواز سیستم

پس از اتمام موفقیت‌آمیز اسکریپت و مشاهده پیام سبز رنگ فعال‌سازی، سرور را یک بار ریستارت کنید تا پروکسی روی تمام لایه‌های سیستم بشیند:

```bash
sudo reboot

```

پس از بالا آمدن سرور (حدود ۱ الی ۲ دقیقه بعد)، دوباره به سرور SSH بزنید و دستور زیر را تست کنید:

```bash
sudo apt update

```

خواهید دید که مخازن اوبونتو و داکر بدون هیچ تحریم، فیلترینگ یا تاخیری، با بالاترین سرعت ممکن لود می‌شوند.
